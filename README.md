# Анализ пролонгаций договоров | Data Analyst Test Task

Проект анализирует пролонгации договоров за 2023 год, рассчитывая специальные коэффициенты, которые показывают связь между отгрузками и пролонгациями проектов.

## 📋 Описание

Данное тестовое задание решает задачу анализа поведения пролонгируемых договоров:

- **Коэффициент 1**: Доля отгрузки за текущий месяц проектов, пролонгированных из предыдущих месяцев
- **Коэффициент 2**: Доля отгрузки за текущий месяц проектов, которые «вернулись» после пропуска месяца пролонгации

Результаты сохраняются в формате Excel с детализацией по месяцам.

## 📊 Логика расчётов

### Коэффициент 1 (Продолжающиеся пролонгации)

```
Коэфф_1(T) = Отгрузка(T) проектов завершённых в T-1 и пролонгированных в T
             ─────────────────────────────────────────────────────────────
             Отгрузка(T-1) всех проектов, завершённых в T-1
```

### Коэффициент 2 (Возвращающиеся пролонгации)

```
Коэфф_2(T) = Отгрузка(T) проектов завершённых в T-2, НЕ пролонгированных в T-1, но пролонгированных в T
             ───────────────────────────────────────────────────────────────────────────────────────
             Отгрузка(T-2) проектов завершённых в T-2 и НЕ пролонгированных в T-1
```

Где T — целевой месяц анализа.

## 📝 Входные данные

### prolongations.csv

| Поле | Описание |
|------|---------|
| project_id | Уникальный идентификатор проекта |
| completion_date | Дата завершения проекта (формат: YYYY-MM) |
| prolongation_date | Дата пролонгации договора (формат: YYYY-MM) |
| amount | Сумма отгрузки |

### financial_data.csv

| Поле | Описание |
|------|---------|
| project_id | Уникальный идентификатор проекта |
| completion_month | Месяц завершения (формат: Месяц YYYY) |
| shipment_value | Сумма отгрузки |
| status | Статус проекта |

## 📤 Выходные данные

Результат сохраняется в **`Prolongation_Report_2023.xlsx`** с листами:

- **Summary**: Сводная таблица с коэффициентами по месяцам
- **Details**: Детальный расчёт с промежуточными значениями
- **Metadata**: Информация о датах расчётов и версии

## 🔍 Примечания

- Анализ фокусируется на **2023 году**
- Данные обрабатываются **батчами по 10,000 строк** для оптимизации памяти
- Российские названия месяцев автоматически преобразуются в формат YYYY-MM
- Все даты нормализуются к формату ISO 8601

## ⚠️ Известные ограничения

- Требуется наличие обоих CSV файлов в папке `data/`
- Пустые или некорректные даты пропускаются с предупреждением
- Минимально требуется 3 месяца данных для корректного расчёта Коэффициента 2
